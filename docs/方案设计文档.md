# 方案设计文档

## 1. 项目目标

本项目首版目标是跑通一条最核心的 AI 学习闭环：

用户输入学习内容 -> AI 生成题目 -> 用户闯关答题 -> AI 生成学习分析报告 -> 分享学习结果。

首版不优先做登录注册、复杂用户体系、多人 PK、排行榜、企业后台和大型知识库，先验证用户是否愿意输入内容、连续答题、查看复盘报告。

## 2. 当前已落地版本

当前版本已完成 P0 + 基础分享闭环：

- 文本输入创建学习会话。
- 后端生成 5 到 10 道单选题/判断题。
- 用户逐题答题并获得即时对错反馈和解析。
- 完成答题后生成复盘报告。
- 前端展示基础分享结果页。
- 后端保存学习会话、输入材料、题目、答题记录和学习报告。
- H5 开发环境已通过 Playwright 端到端验证。

当前版本暂不实现网页解析、文件上传解析、视频解析、学习记录中心、错题本和二刷练习。

## 3. MVP 核心流程

### 3.1 用户输入内容

当前落地版本只实现文本输入。链接、文件、视频入口可在前端展示，但不会进入解析流程。

后续输入能力优先级：

1. 文本输入
2. 网页链接
3. 文件上传
4. 视频链接

### 3.2 AI 生成题目

后端接收用户输入后，调用 AI provider 生成 5 到 10 道题。

题目首版支持：

- 单选题
- 判断题

每道题包含：

- 题干
- 选项
- 正确答案
- 答案解析
- 知识点标签
- 难度

### 3.3 用户答题

用户逐题答题，每次提交答案后立即返回：

- 是否正确
- 正确答案
- 答案解析
- 当前进度

同一学习会话中的同一道题只允许提交一次。

### 3.4 AI 生成报告

用户完成题目后，后端根据题目、答案和错误记录生成复盘报告。

报告包含：

- 本次学习摘要
- 正确率
- 掌握情况
- 错题汇总
- 薄弱知识点
- 下一步学习建议

### 3.5 分享

首版支持基础分享学习结果页，展示：

- 学习主题
- 得分
- 正确率
- 简短学习总结

## 4. 技术选型

### 4.1 小程序端

前端采用 Taro 4.2 + React。

当前前端实现采用单页状态流转，页面状态包括：

- 首页输入
- AI 生成中
- 闯关答题
- 即时讲解
- 复盘报告
- 分享预览

开发环境接口地址默认 `http://localhost:8080`，可通过 `TARO_APP_API_BASE` 覆盖。

### 4.2 后端

后端采用：

- Java 17+
- Spring Boot 3.5
- Spring MVC
- MyBatis-Plus
- MySQL 8
- Spring AI Alibaba
- 通义千问
- Maven

当前测试环境使用 H2 和 `test` profile，接口测试使用 MockMvc。

### 4.3 AI 调用方案

小程序不直接调用大模型，后端统一封装 AI 调用。

当前 AI provider 支持：

- `mock`：默认 provider，用于本地开发、自动化测试和端到端验证。
- `dashscope`：通过 Spring AI Alibaba ChatClient 调用通义千问，使用结构化 JSON 输出。

调用链路：

```text
Taro 微信小程序/H5
  |
  | HTTP
  v
Spring Boot API 服务
  |
  | SQL
  v
MySQL
  |
  | HTTPS
  v
Spring AI Alibaba
  |
  | HTTPS
  v
通义千问
```

## 5. 后端模块

后端按功能拆分：

- LearningSession：学习会话
- SourceMaterial：输入材料
- Question：题目
- AnswerRecord：答题记录
- LearningReport：学习报告
- AiClient：AI 调用抽象
- MockAiClient：本地 mock AI
- DashScopeAiClient：通义千问 AI

题目生成失败或 JSON 校验失败时，业务层自动重试一次。

## 6. 后端接口设计

接口统一返回格式：

| 字段 | 类型 | 说明 |
|---|---|---|
| code | string | `OK` 或错误码 |
| message | string | 返回信息 |
| data | object/null | 业务数据 |

### 6.1 创建学习会话

```http
POST /api/learning/sessions
```

请求字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| inputType | string | 是 | 当前仅支持 `text` |
| content | string | 是 | 文本内容 |

返回 `data`：

| 字段 | 类型 | 说明 |
|---|---|---|
| sessionId | long | 学习会话 ID |
| status | string | 会话状态 |

### 6.2 生成题目

```http
POST /api/learning/sessions/{sessionId}/questions
```

返回 `data`：

| 字段 | 类型 | 说明 |
|---|---|---|
| sessionId | long | 学习会话 ID |
| questions | array | 题目列表 |

题目字段包括 `id`、`type`、`stem`、`options`、`correctAnswer`、`explanation`、`knowledgePoint`、`difficulty`、`sortOrder`。

### 6.3 提交答案

```http
POST /api/learning/sessions/{sessionId}/answers
```

请求字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| questionId | long | 是 | 题目 ID |
| userAnswer | string | 是 | 用户答案 |

返回 `data`：

| 字段 | 类型 | 说明 |
|---|---|---|
| correct | boolean | 是否正确 |
| correctAnswer | string | 正确答案 |
| explanation | string | 答案解析 |
| progress | object | 已答题数、题目总数和正确数 |

### 6.4 生成报告

```http
POST /api/learning/sessions/{sessionId}/report
```

返回 `data`：

| 字段 | 类型 | 说明 |
|---|---|---|
| reportId | long | 报告 ID |
| summary | string | 学习摘要 |
| mastery | string | 掌握情况 |
| score | int | 得分 |
| accuracy | decimal | 正确率 |
| weakPoints | array | 薄弱知识点 |
| suggestions | array | 学习建议 |
| wrongQuestions | array | 错题汇总 |

### 6.5 查询学习会话

```http
GET /api/learning/sessions/{sessionId}
```

返回 `data`：

- `session`
- `material`
- `questions`
- `answers`
- `report`

## 8. AI 生成方案

AI 返回必须是结构化 JSON。

题目 JSON 包含：

- type
- stem
- options
- correctAnswer
- explanation
- knowledgePoint
- difficulty

后端校验：

- 题目数量是否为 5 到 10 道
- 每道题是否有题干、正确答案和解析
- 单选题是否至少有 4 个选项
- 判断题是否有选项
- 正确答案是否存在于选项中

答案判断不调用 AI，后端直接根据 `correct_answer` 判断。

## 10. 风险与应对

### 10.1 题目质量不稳定

- 固定结构化提示词
- 限制题型
- 后端校验 JSON
- 不合格结果自动重试

### 10.2 AI 成本过高

- 首版限制输入长度
- 限制题目数量
- 报告只在完成答题后生成
- 已生成题目重复请求时直接返回已有题目

### 10.3 长文档解析困难

- 当前版本暂不做长文档解析
- 后续再支持文件大小、类型和长内容摘要策略

### 10.4 小程序审核风险

- 后续增加内容安全检查
- 后续对 AI 输出做敏感词过滤

### 10.5 无登录导致数据归属弱

- 当前版本用 `sessionId` 完成核心流程
- 后续再接入微信登录绑定历史数据

### 10.6 H5 开发跨域

- 后端通过全局 CORS 配置允许本地开发端口访问 `/api/**`
- 当前允许 `localhost:10086`、`127.0.0.1:10086`、`localhost:10087`、`127.0.0.1:10087`

## 11. 验收标准

- [x] 用户不登录也能完成一次完整学习流程。
- [x] 用户输入一段文本后能生成 5 到 10 道题。
- [x] 用户每答一题都能看到对错和解析。
- [x] 用户完成全部题目后能看到复盘报告。
- [x] MySQL 能完整保存学习会话、题目、答题记录和报告。
- [x] AI 返回异常时有失败提示和重试机制。
- [x] 小程序可以分享学习结果页。
- [x] H5 开发环境端到端流程通过 Playwright 验证。

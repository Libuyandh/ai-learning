# 联网搜索工具调用归档

## 功能目标

解决 AI 生成题目时对新知识、冷门概念、多义词理解跑偏的问题。

当前实现使用 Spring AI Alibaba / Spring AI tool calling，在题目生成前允许大模型主动调用 `webSearch` 工具获取联网资料，再基于搜索结果生成题目。

## 实现范围

- 新增 `WebSearchTools.webSearch(String query)` 工具方法。
- 新增 `WebSearchClient` 抽象。
- 新增 `FirecrawlWebSearchClient`，通过 Firecrawl `/v2/search` 获取联网资料。
- 新增 `MockWebSearchClient`，用于本地开发和测试。
- `DashScopeAiClient.generateQuestions()` 接入 `.tools(webSearchTools)`。
- AI 输出改为 `BeanOutputConverter<AiQuestionResponse>` 结构化解析。
- `AiQuestion`、`Question`、`QuestionDto` 增加来源、证据和置信度字段。
- 题目入库前校验 `sourceUrl`、`evidence`、`confidence`。
- 前端答题解析区展示证据和来源链接。

## 后端链路

```text
用户输入学习主题
  |
  v
POST /api/learning/sessions/{sessionId}/questions
  |
  v
DashScopeAiClient.generateQuestions()
  |
  v
Spring AI tool calling
  |
  v
WebSearchTools.webSearch()
  |
  v
WebSearchClient
  |
  v
Firecrawl /v2/search 或 MockWebSearchClient
  |
  v
AI 基于搜索资料生成结构化题目
  |
  v
LearningService 校验并保存题目
```

## 配置

生产环境：

```yaml
search:
  provider: firecrawl
  max-results: 5
  max-content-length: 12000
  api-key: ${FIRECRAWL_API_KEY:}
```

测试环境：

```yaml
search:
  provider: mock
```

## 数据变更

`question` 表新增字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| source_url | varchar(1024) | 题目来源链接 |
| evidence_text | text | 支撑答案的证据片段 |
| confidence | decimal(4,3) | 题目置信度 |

题目接口新增字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| sourceUrl | string | 来源链接 |
| evidence | string | 证据片段 |
| confidence | number | 置信度 |

## 校验规则

- 题目数量必须为 5 到 10 道。
- 题干、正确答案、解析不能为空。
- 选项数量不能少于 2。
- 单选题至少 4 个选项。
- 正确答案必须存在于选项中。
- `sourceUrl` 不能为空。
- `evidence` 不能为空。
- `confidence` 必须在 `0.6` 到 `1` 之间。
- 校验失败后自动重试一次。
- 重试后仍失败，返回 `AI_GENERATION_FAILED`。

## 测试覆盖

已补充 MockMvc 测试：

- `Harness Engineering` 这类新知识输入会触发联网搜索工具。
- 生成题目包含 `sourceUrl`、`evidence`、`confidence`。
- 同一会话重复生成题目时直接返回缓存结果，不重复调用搜索。
- 搜索无结果时拒绝生成幻觉题目，并返回 `AI_GENERATION_FAILED`。
